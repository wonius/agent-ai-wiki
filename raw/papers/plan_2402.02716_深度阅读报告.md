## 2. 一句话概括

首篇系统性综述LLM-based agents的规划能力，提出五分类法：任务分解、规划选择、外部模块、反思与记忆。

---

## 3. 科学问题

### 核心问题
LLM作为自治Agent的规划模块时，如何有效提升其规划能力？

### 背景与重要性
- 规划是Agent最关键能力之一，需要理解、推理和决策
- 传统符号方法需人工转换，容错性差
- RL方法需大量环境交互样本，成本高
- LLM的出现为Agent规划带来新范式

### 现有方法不足
- 缺乏对LLM Agent规划能力的系统性分析
- 现有综述覆盖decision-making、reasoning、tool learning，未深入规划能力
- 需要系统的分类框架来组织现有工作

## 4. 技术路线

### 核心技术方案：五分类法
```
                    ┌─────────────────┐
                    │  Task Decomposition │
                    │    (任务分解)      │
                    └────────┬────────┘
                             │
          ┌──────────────────┼──────────────────┐
          │                  │                  │
    ┌────▼────┐      ┌─────▼─────┐     ┌──────▼──────┐
    │Plan     │      │External   │     │Reflection  │
    │Selection│      │Module    │     │& Refinement│
    │(规划选择)│      │(外部模块) │     │(反思改进)  │
    └─────────┘      └──────────┘      └───────────┘
                         │
                    ┌─────▼─────┐
                    │  Memory   │
                    │(记忆增强) │
                    └──────────┘
```

### 五个方向详解

**1. Task Decomposition（任务分解）**
- 将复杂任务拆分为多个子任务
- 方式：Decomposition-First / Interleaved Decomposition
- 代表：CoT, ReAct, HuggingGPT, Plan-and-Solve

**2. Plan Selection（规划选择）**
- 生成多个备选方案，通过搜索算法选择最优
- 代表：ToT (Tree of Thoughts), GoT (Graph of Thoughts), CoT-SC

**3. External Module（外部模块）**
- LLM负责任务形式化，外部规划器执行规划
- 代表：LLM+P, LLM+PDDL

**4. Reflection & Refinement（反思改进）**
- 让LLM反思失败并 refine 规划
- 代表：Reflexion, CRITIC, Self-Refine

**5. Memory-augmented Planning（记忆增强）**
- 通过外部记忆模块存储常识、经验、知识
- 代表：REMEMBER, MemoryBank

### 实验配置
- 评估基准：4个代表性benchmark
- 覆盖各方向代表性方法

---

## 5. 核心发现

### 主要结论
1. 五个方向各有优势和局限，相互补充
2. Task Decomposition 是最直观有效的方法
3. Reflection类方法对错误恢复很重要
4. Memory��块可显著提升长程任务表现

### 关键数据/指标
- 综述50+相关工作
- 系统分类可追溯的方法

### 消融实验
- 各方向在不同任务类型上的有效性对比

## 6. 创新点

1. **首个系统性综述**：首次全面分析LLM-based agents的规划能力
2. **五分类法**：提出五大研究方向，涵盖主流方法
3. **Benchmark评估**：在4个基准上评估代表性方法

---

## 7. 与相关工作对比

| 方向 | 核心思路 | 优势 | 局限 |
|:---|:---|:---|:---|
| Task Decomposition | 拆分任务逐步解决 | 降低复杂度 | 错误累积 |
| Plan Selection | 多方案选择 | 探索空间大 | 计算开销 |
| External Module | 形式化+外部规划 | 精确 | LLM能力受限 |
| Reflection | 反思改进 | 错误恢复 | 需显式反馈 |
| Memory | 记忆增强 | 利用经验 | 检索质量 |

---

## 8. 局限性

1. **分类边界模糊**：五类方法常混用，如 Decomposition + Reflection
2. **评估不充分**：benchmark覆盖有限
3. **实时性**：快速演进，部分内容可能过时
4. **缺少女工标注数据支撑**

## 10. 对Agent实践的指导

### 可直接借鉴的方法
1. **五分类法框架**：快速定位适合自己场景的方法
2. **组合策略**：多方法叠加（如 Decomposition + Reflection）
3. **外部规划器**：复杂任务可考虑LLM+P模式

### 需要适配的场景
- 长程任务：优先考虑 Memory 增强
- 开放环境：考虑 Plan Selection 探索
- 错误敏感场景：加入 Reflection 机制

### 潜在应用
- Agent系统架构设计指南
- 新方法发现的文献地图
- 选择哪种规划方法的决策树
