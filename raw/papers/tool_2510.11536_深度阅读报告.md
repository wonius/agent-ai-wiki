## 第二部分：一句话概括（≤50字）

**核心贡献**: 设计CodeWatcher系统，通过VS Code插件无侵入捕获CGT（代码生成工具）使用事件，为研究LLM生成代码对开发者行为的影响提供基础设施。

---

## 第三部分：科学问题

### 核心问题
理解开发者如何与代码生成工具（CGT）交互是困难的，缺乏不影响工作流的高质量实时交互数据收集工具。

### 背景与重要性
- CGT（如GitHub Copilot、Amazon Q）在IDE中生成代码广泛使用
- 现有评估方法（测试用例、静态分析）只看结果，不了解过程
- 实时形成性反馈对学习编程很重要
- 大规模在线场景难以提供个性化反馈
- 传统工具（AutoGrader, Coding Rooms）关注正确性/语法，忽略行为指标
- 现有研究多为调查、视频、访谈，缺乏实时IDE数据

### 现有方法不足
- 现有工具缺乏实时性、依赖批量分析
- 大多数工业工具专有，仅关注聚合指标
- 无法区分LLM生成代码 vs 人类编写代码
- 无法捕获混淆、重写、盲目复制粘贴等行为

## 第四部分：技术路线

### 核心技术方案

1. **事件捕获系统**:
   - VS Code插件（客户端）
   - Python RESTful API（服务器）
   - MongoDB后端
   - Docker容器化部署

2. **捕获的8种事件类型**:
   | 事件 | 描述 | 捕获字段 |
   |:---|:---|:---|
   | Start | 新文件打开 | Time |
   | End | 文件失去焦点 | Time |
   | Insertion | 代码插入（>3字符，过滤噪音） | Time, Text, Line |
   | Deletion | 代码删除 | Time, Text, Line |
   | Focus | 获得焦点 | Time |
   | Unfocus | 失去焦点 | Time |
   | Copy | 复制命令 | Time, Text |
   | Paste | 粘贴命令 | Time, Text |

3. **设计原则**:
   - 轻量级、无侵入
   - 最小开销（依赖原生编辑器hooks）
   - 过滤字符级输入，聚合为高级事件
   - 区分粘贴输入 vs LLM代码接受

4. **系统架构**:
   - 客户端：VS Code插件（JavaScript）
   - 服务器：FastAPI (Python)
   - 数据库：MongoDB（灵活schema）
   - 通信：RESTful, JSON over HTTP

### 验证方法
- 150次试验，2用户模拟真实开发者工作流
- 精确匹配验证：事件类型、Text、Line字段
- 时间戳验证：与外部记录对比

## 第五部分：核心发现

### 验证结果
| 场景 | 事件类型 | 精确度 | 召回率 |
|:---|:---|:---:|:---:|
| Insertion | Insertion | 1.00 | 1.00 |
| Deletion | Deletion | 1.00 | 1.00 |
| Copy-Paste | Copy, Paste | - | - |
| Focus Switch | Focus, Unfocus | 0.87 | 1.00 |
| Back-and-Forth | Insertion, Deletion | 1.00 | 1.00 |
| Multiple Files | Start, Insertion | 1.00 | 1.00 |
| Under Load | All | 0.93 | 1.00 |
| Concurrent Users | All | 1.00 | 1.00 |

### 关键发现
- 系统能准确捕获各类交互事件
- 精确度和召回率均高（多数达到1.00）
- Focus切换精确度略低（0.87），但可接受
- 负载和并发测试表现稳健

## 第六部分：创新点（≤3条）

1. **无侵入IDE数据收集**: 首个轻量级捕获CGT交互的完整系统
2. **事件语义丰富**: 捕获代码内容、行号、时间戳，支持会话重建
3. **实时干预支持**: 规则引擎支持基于行为数据的实时反馈

## 第八部分：局限性

1. **VS Code专属**: 当前仅支持VS Code
2. **事件过滤**: 插入>3字符的要求可能遗漏短代码接受
3. **隐私考虑**: 收集开发者行为数据需考虑隐私
4. **规则引擎**: 论文未详述规则引擎实现
