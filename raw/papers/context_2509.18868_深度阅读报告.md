## 2. 一句话概括

提出LLM记忆的完整治理框架：四元分类法（参数/上下文/外部/程序记忆）+ 三设置评估协议 + DMM-Gov动态更新机制。

## 3. 科学问题

**核心问题**：LLM如何存储、更新、遗忘知识？如何系统评估和治理？

**背景与重要性**：
- 从"语言理解"到"知识应用"的转变核心是记忆机制
- 医疗/金融法律场景需要准确、最新的知识
- 现有研究边界模糊、评估碎片化、偏见严重

**现有方法不足**：
- 概念边界模糊：参数/上下文/外部记忆定义混乱
- 评估碎片化：RAG系统缺乏忠诚度评估
- LLM-as-Judge偏见：位置/顺序/自我偏好

## 4. 技术路线

### 核心技术方案

#### 1. 记忆四元分类
- **位置-持久性-写入/访问路径-可控性**
- parametric, contextual, external, procedural/episodic

#### 2. 三设置评估协议
- 参数仅(PO)：封闭书本式召回
- 离线检索：RAG设置
- 在线检索：实时信息

#### 3. 分层评估
- 参数记忆：封闭书召回、编辑差分、隐私
- 上下文记忆：位置曲线、中间下降
- 外部记忆：正确性vs归属/忠诚度
- 程序/情景记忆：跨会话一致性、时间线回放

#### 4. DMM-Gov动态治理
- 协调DAPT/TAPT, PEFT, 模型编辑(ROME/MEND/MEMIT/SERAC), RAG
- 循环： admission → rollout → monitoring → rollback → audit

### 实验配置
- 评估基准：多个记忆类型数据集
- 时间对齐协议：时效敏感场景

## 5. 核心发现

### 主要结论
1. 统一治理框架可连接机制-评估-治理全链条
2. 三设置协议避免异构设置下的扭曲比较
3. 四类可测试命题指导实践

### 关键数据
- 位置曲线：中间位置性能下降
- 参数vs上下文记忆边界
- 编辑技术的Pareto分析

## 6. 创新点

1. **统一的记忆定义和四元分类法**
2. **三设置评估协议**消除设置异构性
3. **DMM-Gov可审计治理循环**
4. **四类可测试命题**：最小可识别性、评估卡片、因果约束编辑、检索vs长上下文边界
