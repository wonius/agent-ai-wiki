## 2. 一句话概括

提出AgentGen框架，利用LLM自动生成多样环境+双向演化任务难度，显著提升LLM规划能力，Llama-3.1-70B超越GPT-4。

---

## 3. 科学问题

### 核心问题
如何自动化生成足够多样和大规模的规划任务数据，用于提升LLM Agent的规划能力？

### 背景与重要性
- Agent训练依赖专家级轨迹数据（action-observation序列）进行指令微调
- 现有工作依赖人工设计环境和任务，成本高且多样性不足
- 缺乏大规模、多样化的训练数据限制了LLM规划能力提升

### 现有方法不足
- 手动设计环境和任务需领域专家参与
- 环境和任务数量有限（通常仅几个）
- 任务难度缺乏渐进式分布

---

## 4. 技术路线

### 核心技术方案：AgentGen 两阶段框架
```
┌─────────────────────────────────────────────────────────────────┐
│                      AGENTGEN 流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌──────────────────┐      ┌──────────────────────────────┐  │
│   │ Inspiration      │      │        Environment            │  │
│   │ Corpus           │────▶ │        Generation            │  │
│   │ (领域文本语料)    │      │        (环境生成)             │  │
│   └──────────────────┘      └──────────────────────────────┘  │
│                                               │                 │
│                                               ▼                 │
│   ┌──────────────────────────────────────────────────────────┐  │
│   │                   Bi-Evol                                 │  │
│   │  ┌─────────┐    ┌─────────┐    ┌─────────┐              │  │
│   │  │ Easier  │◀───│  Seed   │───▶│ Harder  │              │  │
│   │  │ (简化)  │    │ (初始)  │    │ (加难)  │              │  │
│   │  └─────────┘    └─────────┘    └─────────┘              │  │
│   │        双向演化 → 平滑难度曲线                              │  │
│   └──────────────────────────────────────────────────────────┘  │
│                              │                                   │
│                              ▼                                   │
│   ┌──────────────────────────────────────────────────────────┐  │
│   │            7246条高质量轨迹 → 指令微调LLM                 │  │
│   └──────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 关键模块

**1. 环境生成 (Environment Generation)**
- 使用灵感语料库（Inspiration Corpus）：多样化领域文本片段作为上下文
- LLM生成环境规格说明（自然语言描述）
- LLM生成对应代码（Python/PDDL）
- 迭代扩展环境库

**2. 任务生成 (Task Generation)**
- 双向演化方法 Bi-Evol：
  - 简化方向：减少约束，降低难度
  - 加难方向：增加约束，提高难度
- 生成平滑难度曲线的任务集

### 实验配置
- 生成：**592个环境，每个20个任务**
- 轨迹：7246条高质量轨迹
- 评估：AgentBoard
- 模型：Llama-3.1-8B, Llama-3.1-70B

---

## 5. 核心发现

### 主要结论
1. AgentGen **10倍提升** Llama-3.1-8B 在域内任务表现（33.3 vs 3.0）
2. **Llama-3.1-8B 超越 GPT-3.5** 整体性能
3. **AgentGen-tuned Llama-3.1-70B 超越 GPT-4**，达到SOTA
4. 域外任务同样显著提升：8B提升10.0%，70B提升3.7%

### 关键数据/指标
- 592环境 × 20任务 = 11840规划任务
- 7246条专家轨迹
- 域内：33.3 (提升10倍)
- 域外：零样本评估

### 消融实验
- 验证灵感语料库多样性对环境生成的影响
- 验证Bi-Evol对难度分布平滑性的贡献

## 6. 创新点

1. **首个自动化框架**：自动生成多样环境+规划任务，用于LLM Agent训练
2. **灵感语料库方法**：利用领域文本提升环境多样性
3. **双向演化Bi-Evol**：生成平滑难度曲线的任务集

---

## 7. 与相关工作对比

| 方法 | 核心思路 | 优势 | 局限 |
|:---|:---|:---|:---|
| 手工设计环境 | 人工定义场景 | 质量高 | 成本高、多样性低 |
| 少量环境+微调 | 有限数据 | 简单 | 泛化差 |
| **AgentGen** | 自动生成 | 大规模、多样性 | 依赖LLM质量 |

---

## 8. 局限性

1. **依赖LLM质量**：生成环境和任务的LLM能力直接影响数据质量
2. **PDDL聚焦**：主要生成PDDL格式任务，域外泛化仍需验证
3. **评估局限**：仅在AgentBoard上验证
4. **灵感语料库选择**：决定环境覆盖范围

## 10. 对Agent实践的指导

### 可直接借鉴的方法
1. **自动数据合成**：用LLM生成训练数据，减少人工标注
2. **难度渐进**：双向演化确保学习曲线平滑
3. **灵感触发**：使用领域相关文本提升生成多样性

### 需要适配的场景
- 低资源场景：无足够标注数据时
- 新领域快速部署：生成领域特定任务
- 持续学习：新工具/环境迭代

### 潜在应用
- Agent训练数据自动扩增
- 评测基准自动生成
- 多领域规划能力快速适配
