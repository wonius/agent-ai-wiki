## 一、研究背景与问题

### 1.1 会话搜索的核心挑战

会话搜索系统（如对话式搜索引擎）要求用户能以自然的多轮对话形式搜索信息。与传统搜索不同，用户的后续 query 通常：

- **包含指代**：如"它"、"那个"、"上面的结果"
- **存在省略**：如"看看第二个"
- **具有歧义**：如"苹果"可能是水果也可能是公司

这些 query 如果直接用于检索，效果极差。**Conversational Query Reformulation (CQR)** 正是解决这个问题的技术——将不完整的、含指代的会话 query 转换为独立、完整、可直接检索的形式。

### 1.2 现有 CQR 方法的两大致命缺陷

**缺陷一：过度依赖外部监督**

现有方法需要三种监督之一：
- 人工重写（Human Rewrite）：成本极高，不可规模化
- 大模型蒸馏（Distillation）：需要 GPT-4 等大模型，成本高且引入额外依赖
- 弱监督信号：效果差

这些外部监督方法存在共同问题：**外部重写结果不一定与下游检索器的偏好对齐**。

**缺陷二：重写模型与检索器对齐不足**

即使有了高质量的重写，**重写模型和检索器是独立训练的**，两者之间没有直接的优化信号传递。重写质量的标准是"像人类"，而不是"能让检索器找到正确答案"。

这导致一个尴尬的现象：人类觉得"好的重写"，检索器并不一定认可。

### 1.3 论文的核心贡献

ConvSearch-R1 首次提出**完全摆脱外部重写监督**的框架——用**检索信号本身**驱动 RL，直接优化"能让检索器找到正确答案的重写策略"。

## 二、核心方法

### 2.1 核心洞察：用检索信号替代外部监督

论文的关键洞察是：**检索结果本身就是最直接的监督信号**。

如果一个重写后的 query 能让检索器找到正确答案，说明重写质量高；如果找不到，说明重写有问题。检索器返回的排名（ranking position）包含了丰富的信息——不只是 binary success/fail，而是一个连续的、精细的反馈信号。

因此，**不需要人类重写，不需要 GPT-4 做蒸馏，直接用检索结果训练重写模型**。

### 2.2 两阶段 Pipeline

```
Stage 1: Self-Driven Policy Warm-Up (SDPWU)
    → 解决 RL 冷启动问题
    → 生成具备基本能力（格式、推理、重写）的初始策略

Stage 2: Retrieval-Guided Reinforcement Learning (RGRL)
    → 用 RIRS reward 直接优化重写策略
    → GRPO 算法，无需 reward model
```

### 2.3 Stage 1: SDPWU — 自驱动策略热身

#### 为什么需要 SFT？

论文引用了 Chu et al. (2025) 的发现：RL 训练需要模型先具备**稳定的输出格式遵循能力**，否则 RL 探索空间过大难以收敛。因此需要先用 SFT 给模型一个好的起点。

但之前的方法依赖外部 LLM（GPT-4）做知识蒸馏，成本高且可能引入领域不匹配。SDPWU 的目标是**完全不依赖外部监督**。

#### 自蒸馏流程

**Step 1 — 自生成（Self-Distillation）**

用 few-shot prompting 让初始模型 π_init 自己生成重写结果：
```
D^d = { y_i = π_init(x_i, ρ, instruction) | x_i ∈ D }
```
ρ 是固定的 few-shot 示例，D 是原始数据集。

**Step 2 — 格式过滤**

过滤掉格式不合规的输出：
```
D^f = { y ∈ D^d | g(y) = 1 }
```
g(y) 是格式验证函数。

**Step 3 — 高质量样本筛选（关键创新）**

只保留那些"重写后检索结果中 gold passage 排在第1位"的样本：
```
SD-DATA = { [c, q, r] | rewrite 让 gold passage rank=1 }
```
每个样本是三元组：对话历史 c、当前 query q、推理-重写对 r。

这就是"自驱动"的含义：**不需要外部强大的 LLM 来判断什么是好重写，检索器本身就是裁判**。

**Step 4 — SFT**

在 SD-DATA 上做 SFT，最大化 ground-truth 输出似然：
```
π_SFT = SFT(π_init, SD-DATA)
```

经过 SFT 后，模型具备：稳定格式遵循能力、生成推理链的能力、基础 query 重写能力。

### 2.4 Stage 2: RGRL — 检索信号驱动的强化学习

#### 传统方法的奖励稀疏问题

如果直接用检索指标（MRR@3、NDCG@3）作为奖励，由于大部分 query 的得分都是 0（binary），会导致**严重的奖励稀疏**，RL 难以有效学习。

论文的实验数据（图8）证实了这一点：直接用检索指标做奖励，RL 几乎无法学习。

#### Rank-Incentive Reward Shaping (RIRS) — 核心创新

RIRS 的设计思想：**将离散的排名转化为连续的奖励信号**。

核心公式（分段函数）：
```
R(ξ) = f_{[1,10]→[1,2]}(ξ) · I_{[1,10]}(ξ)
     + f_{(10,100]→[0,1]}(ξ) · I_{(10,100]}(ξ)
```

其中 ξ 是排名位置。翻译成人话：
- **排名 1-10**：给予 1~2 的高奖励，排名越高奖励越高（非线性映射）
- **排名 11-100**：给予 0~1 的次级奖励
- **排名 > 100**：奖励为 0

**为什么这样设计？**

1. **用户的注意力集中在顶部结果**：排名第5和第6的差异对用户体验影响不大，但排名第1和第5差异巨大——RIRS 反映了这种用户注意力分布
2. **保留语义阈值**：不把所有高分都映射到相同值，而是保留了相对差异
3. **密集反馈**：即使没有排到前10，也有正反馈（0.5-1.0 的奖励），RL 能持续学习

加上格式合规性后的完整奖励：
```
R(ξ, φ) = R(ξ) · I(φ=1) + δ · I(φ=0)
```
格式不合规 → δ = -0.1 惩罚。

#### GRPO 算法

ACON 采用 GRPO（Group Relative Policy Optimization，Shao et al., 2024）作为 RL 算法。GRPO 是一种不需要显式 reward model 和 value model 的高效 RL 算法。

优化目标：
```
J_GRPO(θ) = E[ Σ_i min(π_θ(r_i|x)/π_θ_old(r_i|x) · A_i, clip(...)·A_i) - β·D_KL(π_θ || π_SFT) ]
```
其中 A_i 是用 RIRS 奖励标准化的 advantage，D_KL 惩罚确保策略不会偏离 SFT 学到的 π_SFT 太远。

### 2.5 为什么 3B 能超越 7B？

现有 SOTA 方法（如 RETPO、AdaQR）使用 DPO（Direct Preference Optimization）来对齐重写模型和检索器。但 DPO 需要额外的 reward model 或偏好数据。

ConvSearch-R1 使用 RL + RIRS，允许模型**通过试错探索**来发现比人类重写更好的策略。DPO 则受限于"人类偏好"的上限。

**关键差异：DPO 是模仿学习，RL + RIRS 是探索学习。**

实验结果：Llama3.2-3B（3B 参数）显著超越依赖更大模型监督的 7B 竞品，证明了 RL 探索策略的有效性。

## 三、实验结果

### 3.1 主要结果

**TopiOCQA 数据集（最具挑战性）：**

| 模型 | MRR@3 | NDCG@3 | R@20 |
|------|-------|--------|------|
| ConvSearch-R1 (Llama3.2-3B) | **SOTA** | **SOTA** | **SOTA** |
| RETPO (Llama2-7B) | 低于本文 | 低于本文 | 低于本文 |
| AdaQR (Llama2-7B) | 低于本文 | 低于本文 | 低于本文 |

ConvSearch-R1 在 TopiOCQA 上**超越 SOTA 超过 10%**。

### 3.2 泛化能力

论文测试了跨数据集泛化（训练在 TopiOCQA，评估在 QReCC）：

ConvSearch-R1 展现出**显著更好的泛化能力**，归因于 RIRS reward 鼓励模型通过试错接触更多样化的高质量数据。

### 3.3 关键发现

**发现一：人类重写并不总是最优的**
- 人工重写不仅标注成本高，而且重写结果与检索器偏好不对齐
- 有时人工重写还不如模型自己的原始 query

**发现二：直接用 Reasoning 模型（如 DS-R1-Distill-Qwen-7B）做 CQR 效果很差**
- 通用域的推理能力不等于会话搜索的重写能力
- 即使有长链思考能力，R1 类模型也无法在 CQR 场景超越本文方法

**发现三：检索信号 >> 人类重写作为监督**
- 方法仅依赖检索信号，无需任何外部监督
- 但效果远超依赖外部 LLM 蒸馏的方法

| 类别 | 代表方法 | 与 ConvSearch-R1 的关系 |

| 无重写优化 | Raw Query, Human Rewrite | Baseline，凸显 CQR 的必要性 |

**ConvSearch-R1 的核心突破：** 是首个**完全不依赖外部重写监督**的方法，也是首个用**RL + RIRS** 替代 DPO/DPO-style 偏好的方法。

## 五、局限性

1. **仅验证了 CQR 任务**：方法设计围绕"重写 query 以适配检索器"，对其他 Agent 任务（如工具调用、多步推理）的泛化性未知
2. **检索器不训练**：论文用 BM25 和 ANCE 但不训练检索器，检索器能力上限成为系统瓶颈
3. **RIRS 的超参数**：分段奖励的区间划分（1-10, 11-100）和映射函数设计依赖经验和实验调参
4. **冷启动质量依赖 few-shot**：SDPWU 阶段的效果依赖初始 few-shot prompt 的质量
5. **格式验证函数 g(y) 需要人工定义**：格式合规性的判断标准需要针对不同场景单独设计

## 六、一句话总结各部分

- **问题根源**：CQR 依赖外部监督（人类标注/GPT-4）且重写与检索器不对齐
- **核心洞察**：检索信号本身就是最直接、最免费的监督——检索结果会说话
- **Stage 1（SDPWU）**：检索器筛选自生成高质量数据 → SFT 解决冷启动，无需外部 LLM 蒸馏
- **Stage 2（RIRS + GRPO）**：分段奖励函数解决稀疏问题，GRPO 无需 reward model，直接用检索排名作为连续反馈
- **3B 超越 7B**：RL 探索学习 > DPO 模仿学习，探索空间不受人类偏好上限限制
- **关键启示**：当某个领域的"评估器"（这里是检索器）本身足够强时，可以直接用它作为训练信号的来源，而不需要人工标注

**实际可借鉴之处：**

1. **检索信号 = 免费监督**：如果你的 Agent 任务有明确的环境反馈（如检索器、API 返回值、执行结果），可以用类似思路设计"环境信号驱动的 RL"，减少对人工标注的依赖
2. **RIRS 的奖励塑形思路**：将稀疏的二值奖励塑造成连续的、分段的光滑奖励，对任何 RL 训练场景都有参考价值
3. **SDPWU 的对比筛选**：不需要 ground truth，而是用"任务成功"作为筛选条件自构建训练数据——这个思路可以直接迁移到其他缺乏标注数据的场景
