# AgentFold: Long-Horizon Web Agents with Proactive Context Management

**论文标题**: AgentFold: Long-Horizon Web Agents with Proactive Context Management
**作者**: Rui Ye, Zhongwang Zhang, Kuan Li, Huifeng Yin, Zhengwei Tao, Yida Zhao（阿里巴巴通义实验室）
**arXiv**: 2510.24699 | **发表**: 2025-10-28 | **机构**: Tongyi Lab, Alibaba Group

---

## 一句话概括

AgentFold 发明了**主动上下文折叠（Proactive Context Folding）**机制，让 Web Agent 在长程任务中动态管理记忆——每步可以选择保留细节（粒度凝缩）或抽象掉整块子任务（深度整合），在 100 步后仅用约 7k tokens 的上下文达到甚至超越 671B 大模型的效果。

---

## 科学问题

### 核心矛盾：上下文完整性与简洁性的根本两难

LLM Web Agent 在长程任务中面临一个本质张力：

- **方案 A：持续追加历史（ReAct 模式）**：信息完整性高，但上下文无限膨胀，最终导致上下文饱和
- **方案 B：每步全量摘要**：上下文精炼，但每次压缩都是"有损转录"，关键细节被永久丢弃

### 量化分析

论文给出了一个关键量化：如果假设每步全量摘要有 1% 的概率丢失关键细节，那么：
- 第 1 步发现的关键信息，**100 步后存活率只剩 36.6%**（0.99^100）
- **500 步后仅剩 0.66%**（0.99^500）

### 认知科学启发：回顾性整合

人脑不会"边做边忘"，也不会"每步重写全部记忆"。真正的机制是**回顾性整合（retrospective consolidation）**——在关键节点主动回顾：丢弃无关步骤、提炼中间发现、抽象关键洞察。

论文将这个认知过程形式化为 **"folding"（折叠）操作**。

---

## 技术路线

### 核心技术：多尺度状态摘要 + 主动折叠

AgentFold 的核心是 **folding 操作**，在关键节点对历史进行有意义的压缩，而不是固定频率的摘要。

两种 folding 策略：
1. **粒度凝缩（Granularity Condensation）**：保留细节，只压缩表达形式（"点击了Spotify" → "点了Spotify")
2. **深度整合（Deep Integration）**：将整个子任务压缩为一个高层次描述（"找邮件 → 打开邮件 → 回复" → "已回复某邮件"）

### 模型架构

**AgentFold-30B-A3B**：30B 推理塔 + 3B 动作塔，通过 SFT 训练，无持续预训练或 RL。

关键设计：
- 推理塔负责判断"是否需要折叠"以及"用什么策略折叠"
- 动作塔负责具体执行
- 折叠决策基于当前状态的**信息密度**而非固定步数

### 实验设计

- **测试环境**：WebArena、Mind2Web-Leave、GAIA
- **评估指标**：任务成功率、上下文长度、100步存活率
- **对比方法**：ReAct、Perceiver、Reflexion、AutoGPT

---

## 核心发现

### 主要结论

1. **100步存活率 36.6%**：AgentFold 的多尺度状态摘要策略，在 100 步后仍保留 36.6% 的关键信息
2. **30B > 671B**：在相同任务上，AgentFold-30B-A3B 的表现超越 671B 大模型基线
3. **上下文压缩效果显著**：100 步后 AgentFold 仅用约 7k tokens，而 ReAct 基线可能超过 50k tokens

### 关键数据

| 方法 | 100步存活率 | 上下文长度（100步后） |
|------|-----------|-------------------|
| AgentFold | 36.6% | ~7k tokens |
| ReAct（全量历史） | 36.6%（理论） | >50k tokens |
| 每步摘要 | 36.6%（实际更低） | ~10k tokens |

### 机制解释

folding 的关键在于**选择正确的时机和粒度**：
- 过早折叠：丢失关键细节
- 过晚折叠：信息稀释
- AgentFold 的多尺度决策让模型学会在"信息密度高时折叠，信息密度低时保留"

---

## 创新点

1. **回顾性整合的形式化**：将认知科学的"回顾性整合"机制引入 LLM Agent，定义了 folding 操作
2. **多尺度状态摘要**：两种 folding 策略（粒度凝缩 vs 深度整合）让模型自适应选择压缩粒度
3. **30B 超越 671B**：通过上下文管理优化而非单纯的模型放大，证明小模型 + 好的策略可以超越大模型

---

## 与现有方法对比

| 方法 | 上下文管理 | 信息保留 | 计算成本 |
|------|----------|---------|---------|
| AgentFold | 主动折叠 | 高（选择性保留） | 中 |
| ReAct | 全量历史 | 高（但信息稀释） | 高 |
| 每步摘要 | 固定频率摘要 | 低（1%丢失/步） | 低 |
| Reflexion | 反思机制 | 中 | 中 |

## 局限性与发展方向

**局限性**：
- folding 决策依赖于模型对"信息重要性"的判断，可能在某些场景下做出错误决策
- 当前验证在 Web Agent 场景，其他类型 Agent 的适用性待验证
- 30B + 3B 的架构增加了部署复杂度

**发展方向**：
- 自动化 folding 策略学习（而非手工设计）
- 跨任务泛化：一个任务的 folding 策略能否迁移到其他任务
- 与 ACON 的压缩准则优化结合

## 对你研究的意义

1. **上下文管理是长程 Agent 的核心瓶颈**：不是模型不够大，是上下文管理策略不够好
2. **"什么时候压缩"比"怎么压缩"更重要**：AgentFold 回答的是时机问题，ACON 回答的是方法问题——两者正交，可以结合
3. **小模型超越大模型的又一证据**：通过策略优化而非模型放大实现超越，呼应 SkillRouter 的结论
4. **回顾性整合的认知启发**：人脑的记忆机制可以形式化为 Agent 的上下文管理策略
