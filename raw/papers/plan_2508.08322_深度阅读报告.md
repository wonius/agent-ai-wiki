## 2. 一句话概括

提出多组件上下文工程工作流：Intent Translator + Elicit检索 + NotebookLM合成 + Claude Code多智能体，显著提升代码助手准确率。

---

## 3. 科学问题

### 核心问题
如何解决LLM代码助手在复杂代码库级任务中因上下文限制和知识差距导致的准确性问题？

### 背景与重要性
- LLMs在代码生成和软件工程任务中表现优异
- 复杂多文件项目面临上下文窗口限制和幻觉风险
- 纯prompt驱动的单智能体方案难以处理跨文件协调

### 现有方法不足
- 静态上下文固定不变，无法覆盖所有任务细节
- 单智能体方案在非平凡功能上产生不完整/错误方案
- 缺乏外部知识注入和检索增强

### 关键组件

**1. Intent Translator (GPT-5)**
- 重写用户需求为结构化规格
- 明确化隐式需求
- 分解为步骤清单（如：更新UI组件X、获取后端API数据、调整样式、添加测试）

**2. Semantic Retrieval (Elicit)**
- 语义检索学术论文、文档
- 相关算法、API使用指南
- 外部知识注入

**3. Knowledge Synthesis (NotebookLM)**
- 文档摘要生成
- 可回答详细问题
- 增强上下文理解

**4. Claude Code Multi-Agent**
- 子智能体：Planner, Coder, Tester, Reviewer
- 向量数据库存储代码上下文
- 迭代 refinement

### 实验配置
- 代码库：RainMakerz Next.js (180K行)
- 对比：CodePlan, MASAI, HyperAgent
- 评估：单次成功率、上下文 adherence

## 5. 核心发现

### 主要结论
1. **单次成功率显著提升**
2. 在复杂多文件任务上有效
3. 上下文 adherence 更高
4. 案例：新交互可视化模块一次生成成功

### 关键数据/指标
- 超参数对比：
  - DARS: 47% pass@1
  - HyperAgent: 31.4% (SWE-Bench Verified)
  - MASAI: 28.33%
  - AllianceCoder: 20%

### 消融实验
- 验证各组件贡献
- 验证检索增强效果
