## 2. 一句话概括

提出CUP循环城市规划和多智能体框架，通过Planning-Living-Judging三阶段迭代，实现城市方案的持续生成、评估与优化。

---

## 3. 科学问题

### 核心问题
如何实现城市规划的动态适应性和持续迭代优化，而非一次性静态规划？

### 背景与重要性
- 城市化快速推进，现有城市配置难以满足持续变化的需求
- 传统城市规划依赖静态、一次性干预，缺乏灵活性
- 传统评估方法（观察、问卷调查）费时费力，限制评估频率

### 现有方法不足
- 传统方法（top-down/participatory）缺乏适应性和可扩展性
- 现有ML方法主要针对一次性规划，缺乏迭代闭环
- ABM方法依赖刚性预定义规则

## 4. 技术路线

### 核心技术方案：CUP 三阶段闭环框架
```
┌─────────────────────────────────────────────────────────────────┐
│                  Cyclical Urban Planning (CUP)               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │              Cycle 1 → Cycle 2 → Cycle 3 → ...          │   │
│   └─────────────────────────────────────────────────────────┘   │
│                              ↺                                │
│   ┌───────────────┐     ┌───────────────┐     ┌─────────────┐ │
│   │   Planning    │────▶│    Living     │────▶│   Judging   │ │
│   │   (规划生成)   │     │   (生活模拟)   │     │  (评估反馈)  │ │
│   └───────────────┘     └───────────────┘     └──────────────┘ │
│        │                     │                    │            │
│        └─────────────────────┴────────────────────┘            │
│                    闭环迭代优化                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 关键组件

**1. Planning（规划）**
- Planner Agent：整合专家知识、迭代建议，生成/修订城市规划
- Resident Agents：参与讨论，平衡多元利益
- 迭代公式：
  ```
  P̃k = P.plan(K, Pk-1, Sk-1)  # 修订
  Dk = discuss({Ri}, Pk-1)     # 讨论
  Pk = P.plan(K, P̃k, Sk-1, Dk)  # 整合
  ```

**2. Living（生活模拟）**
- LLM-based ABM：取代传统资源密集型监控
- 移动行为模拟：居民日常轨迹和感受
- 社交行为模拟：社交媒体上的体验传播
- 动态记忆池：记录生活经验

**3. Judging（评判）**
- 定量分析：可达性、生态覆盖等指标
- 定性访谈：居民体验问卷
- 综合评估：生成改进建议

### 实验配置
- 数据集：北京市回龙观社区（3.74km²）
- 模型：GPT-4o（temperature=0）
- 居民Agent：30个
- 迭代轮次：3轮

---

## 5. 核心发现

### 主要结论
1. **Experience指标显著提升**：从65.03% → 69.03%（3轮迭代）
2. **循环有效性**：随迭代增加效果逐步提升
3. 平衡居民需求与规划效果的能力

### 关键数据/指标
| 方法 | 可达性 | 生态 | 体验 |
|:---|:---:|:---:|:---:|
| Random | 61.67 | 50.00 | 65.17 |
| DRL | 66.25 | 56.67 | 64.23 |
| Ours-1st | 65.00 | 73.33 | 65.03 |
| Ours-3rd | 64.17 | 53.33 | **69.03** |

### 消融实验
- 验证迭代轮次对Experience的影响
- 对比定量与定性指标的权衡

## 6. 创新点

1. **CUP新范式**：循环城市持续规划，而非一次性
2. **三阶段闭环**：Planning-Living-Judging动态闭环
3. **LLM-based ABM**：模拟居民行为和社交传播

## 8. 局限性

1. **指标冲突**：定量与定性指标存在权衡冲突
2. **下降趋势**：可达性/生态在迭代中下降
3. **评估不完整**：仅用部分QOL指标
4. **规模限制**：30居民Agent，3轮迭代

## 10. 对Agent实践的指导

### 可直接借鉴的方法
1. **闭环迭代**：规划-执行-评估-改进循环
2. **多角色模拟**：区分Planner/Executor/Judger
3. **动态记忆**：记录状态支持后续决策

### 需要适配的场景
- 需要持续优化的长程任务
- 涉及多方利益协调的决策
- 难以一次性确定最优方案的任务

### 潜在应用
- 智慧城市动态规划
- 企业战略持续迭代优化
- 复杂产品设计流程
