# AutoML-Agent: A Multi-Agent LLM Framework for Full-Pipeline AutoML

**Source**: [arXiv 2410.02958](https://arxiv.org/abs/2410.02958) · 2025-10-10

## Key takeaways

- First LLM-based framework for full-pipeline AutoML (data retrieval → model deployment)
- Retrieval-augmented planning strategy for exploring optimal plans
- Plan decomposition with specialized parallel agents (data/model/deploy)
- Multi-stage verification for code generation quality
- Achieves higher success rate across 7 downstream tasks, 14 datasets

## Core claims

Existing AutoML systems require technical expertise; LLM-based approaches only target specific pipeline steps. AutoML-Agent proposes:

1. **Retrieval-augmented Planning**: Instead of single plan, generate multiple plans based on retrieved knowledge for exploration

2. **Role-specific Decomposition**: Decompose each plan into subtasks (data preprocessing, neural network design) solved by parallel specialized agents

3. **Multi-stage Verification**: Verify executed results and guide code generation LLM

The framework coordinates: Prompt Agent → Data Agent → Model Agent → Deployment Agent, with Agent Manager orchestrating.

## Notable quotes

> This is the first attempt to employ LLMs in a task-agnostic AutoML framework that spans from data retrieval to model deployment.
