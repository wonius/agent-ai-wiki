# A Plan Reuse Mechanism for LLM-Driven Agent

**Source**: [arXiv 2512.21309](https://arxiv.org/abs/2512.21309) · 2024-12-25

## Key takeaways

- AgentReuse mechanism for LLM-driven personal agents (Xiao Ai, Blue Heart V)
- Addresses latency problem: plan generation can take 24+ seconds
- 30% of requests are identical/similar → plan reuse can reduce latency
- Achieves 93% effective reuse rate, reduces latency by 93.12%

## Core claims

LLM-driven agents generate plans via auto-regressive LLM — plan texts are long, causing significant latency (~25 seconds per request). Analysis shows ~30% of requests are **semantically identical or similar**.

**Challenges**:
1. **Similarity definition**: Raw text similarity fails (e.g., "book ticket Hefei→Beijing" vs "Changsha→Shanghai" not recognized as similar)
2. **Key parameter identification**: Need semantic-aware extraction (time, origin, destination)
3. **Effective reuse**: Convert unstructured plan text to structured execution

**AgentReuse** solutions:
- Leverages semantic differences to identify key parameters
- Uses **intent classification** to evaluate request similarity
- Converts plans to structured format for parameter replacement

## Notable quotes

> AgentReuse achieves a 93% effective plan reuse rate and reduces latency by 93.12% compared with baselines without using the reuse mechanism.
