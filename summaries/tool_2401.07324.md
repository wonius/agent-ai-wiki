# Small LLMs Are Weak Tool Learners: A Multi-LLM Agent

**Source**: [arXiv 2401.07324](https://arxiv.org/abs/2401.07324) · 2024-01-14

## Key takeaways

- Decomposes tool-use capability into planner, caller, and summarizer components
- Each component implemented by a separate LLM focusing on specific capability
- Two-stage training: first on entire dataset, then on sub-tasks
- Multi-LLM framework surpasses single-LLM approach in tool-use benchmarks

## Core claims

Traditional approaches train a single LLM with all tool-use capabilities, but smaller models suffer from performance limitations. This paper proposes decomposing the capabilities into three specialized components (planner, caller, summarizer), each fine-tuned on specific sub-tasks, achieving better performance than monolithic approaches.

## Notable quotes

> "The challenge of tool use demands that LLMs not only understand user queries and generate answers accurately but also excel in task planning, tool invocation, and result summarization."
