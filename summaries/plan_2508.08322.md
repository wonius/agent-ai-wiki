# Context Engineering for Multi-Agent LLM Code Assistants

**Source**: [arXiv 2508.08322](https://arxiv.org/abs/2508.08322) · 2025-08-09

## Key takeaways

- Novel context engineering workflow combining Intent Translator, Elicit, NotebookLM, Claude Code
- Integrates intent clarification, RAG, specialized sub-agents
- Achieves higher single-shot success rates on complex repository-level tasks
- Compares with CodePlan, MASAI, HyperAgent

## Core claims

LLMs struggle with complex multi-file projects due to **context limitations and knowledge gaps**. Single static prompts cannot capture all relevant details.

The proposed **Context Engineering** workflow:
1. **Intent Translator** (GPT-5): Rewrite user requests into structured task specifications
2. **Semantic Retrieval** (Elicit): Fetch relevant papers, documentation
3. **Knowledge Synthesis** (NotebookLM): Create summaries for contextual understanding
4. **Claude Code Multi-Agent**: Orchestrate sub-agents (planner, coder, tester, reviewer) with vector database

This addresses failure modes like missing edits in distant files or misusing unfamiliar APIs.

## Notable quotes

> By context engineering, we mean systematically constructing and supplying all relevant information needed for a coding task.
