# Small LLMs Are Weak Tool Learners: A Multi-LLM Agent

**Source**: [arXiv 2401.07324](https://arxiv.org/abs/2401.07324) · 2024-01-14

## Key takeaways

- Multi-LLM agent framework (α-UMi) decomposes tool learning into Planner, Caller, and Summarizer
- Two-stage training paradigm (GLPFT): global fine-tuning → local role-specific fine-tuning
- Smaller LLMs can achieve competitive performance by focusing on single capabilities
- Outperforms single-LLM approaches across various tool-use benchmarks

## Core claims

Traditional approaches train a single LLM with all tool-learning capabilities. This paper reveals that **small LLMs are weak tool learners** and proposes decomposing capabilities into three specialized components:
- **Planner**: generates rationale, decides next action (caller/summarizer/exit)
- **Caller**: invokes specific tools based on planner's rationale
- **Summarizer**: crafts final answer based on execution trajectory

The **GLPFT (Global-to-Local Progressive Fine-tuning)** strategy first fine-tunes on the entire dataset, thencontinually fine-tunes each component on role-specific data.

## Notable quotes

> α-UMi decomposes the capabilities of a single LLM into three components: planner, caller, and summarizer. Each component is implemented by a single LLM and trained to focus on a specific capability.
