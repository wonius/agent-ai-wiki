# 语义压缩

## 定义

语义压缩（Semantic Compression）是相对于底层压缩（如 KV Cache 压缩）而言的压缩层次，指在**语义层面**理解和压缩 Agent 上下文，保留上下文中的关键信息（因果关系、演化状态、前置条件等），而非简单的 token 级别剪枝或量化。

## 压缩层次对比

| 层次 | 代表工作 | 压缩对象 | ACON 的定位 |
|------|---------|---------|------------|
| **KV Cache 压缩** | Zhang et al., 2025 | Attention Key/Value 缓存 | 低级压缩，ACON 是语义级的正交方向 |
| **Token 量化** | Prakash et al., 2024 | 压缩 token 表示 | 精度损失不可控 |
| **检索增强** | Seo et al., 2025 | 上下文窗口内容 | 单步任务，不适合多步 Agent |
| **语义压缩** | **ACON** | 异构上下文信息 | 保留关键语义，支持跨步状态 |

## 为什么 Agent 上下文需要语义压缩？

Agent 上下文本质上充当**世界模型**，包含多种异构信息：

- **因果关系**：例如"邮件留有草稿"
- **演化状态**：例如"账户余额"
- **前置条件**：例如"需要先登录"
- **任务相关决策线索**：例如"截止日期"

简单的 token 级别压缩（量化、剪枝）无法区分这些信息的语义重要性，可能删除关键信号而保留噪声。

## ACON 的语义压缩设计

ACON 同时压缩两类上下文对象：
- **历史压缩**：压缩交互历史（h' = f(h; φ, P_hist)）
- **观察压缩**：压缩环境观察（o' = f(o, h; φ, P_obs)）

压缩后的摘要需要同时满足：
1. 保留关键语义（任务成功）
2. 最小化长度（降低成本）

这通过**压缩准则优化**实现：对比轨迹 → 自然语言反馈 → 更新压缩准则 Prompt。

## 语义压缩 vs 文档压缩

| 维度 | 文档压缩 | 语义压缩（ACON） |
|------|--------|----------------|
| 目标 | 单步问答 | 多步 Agent 任务 |
| 状态保留 | 无需保留（上下文用完即丢） | 必须保留跨步状态 |
| 触发条件 | 文档级压缩 | 阈值触发（T_hist / T_obs） |
