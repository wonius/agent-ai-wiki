# 对比轨迹

## 定义

对比轨迹（Contrastive Trajectory）是 ACON 论文提出的训练信号来源：针对同一任务，同时收集**完整上下文轨迹**（成功）和**压缩上下文轨迹**（失败），通过两者的差异为压缩准则优化提供密集的反馈信号。

## 为什么需要对比轨迹？

传统方法的信号问题是**稀疏**的：
- 二分任务奖励（成功/失败）只在轨迹末端揭示
- 无法直接回答"压缩丢了什么"

对比轨迹把稀疏奖励变成**密集反馈**：
- 同一任务 × 两种上下文 → 直接揭示压缩的失败点
- 每条对比轨迹都能回答"完整成功但压缩失败，压缩到底丢了什么"

## 收集流程

1. 在训练集上同时跑两条轨迹：
   - **H**（完整上下文）：使用全部历史和观察
   - **H'**（压缩上下文）：使用压缩后的历史和观察
2. 收集所有"完整成功但压缩失败"的任务 → **对比子集 D_cont**
3. 对比子集中的每条轨迹都包含了压缩失败的具体原因

## 在 ACON 中的作用

对比轨迹是对比子集 D_cont 的来源，为压缩准则的优化提供自然语言梯度：

```
Feedback = LLM(Feedback Instruction, H, H')
```

这段反馈描述"压缩时丢失了什么关键信息"，即压缩准则的优化方向。

## 与传统 RL 的区别

| | 传统 RL | 对比轨迹 |
|--|--------|---------|
| 信号 | 稀疏二值奖励 | 密集自然语言反馈 |
| 梯度 | 策略梯度（高方差） | 自然语言梯度（可直接理解） |
| 计算 | 需完整 rollout | 直接对比两条轨迹 |
