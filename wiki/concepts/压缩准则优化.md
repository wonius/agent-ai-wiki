# 压缩准则优化

## 定义

压缩准则优化（Compression Guideline Optimization）是 ACON 论文提出的核心方法：将"压缩准则"（即 Prompt）本身当作可优化的对象，在**自然语言空间**通过对比学习信号进行优化，无需梯度、无需更新模型参数。

## 核心思想

传统方法优化压缩模型参数 → 计算量大、需更新权重、对 API 模型不适用。

ACON 的关键洞察：**压缩准则 Prompt 本身就是可优化的资产**，类似于 Textual Gradient Descent / Prompt Optimization 的思想——当无法直接优化模型时，把提示当作可优化的变量。

## 两步交替优化

### Step 1: Utility Maximization（最大化任务奖励）

- 在对比子集（完整成功但压缩失败的任务）上分析失败原因
- 更新压缩准则 Prompt，目标是**保留更多关键信息**
- 提升压缩后的任务成功率

### Step 2: Compression Maximization（最小化上下文成本）

- 仅在压缩后仍然成功的任务上分析"哪些信息实际被用到了"
- 更新压缩准则 Prompt，目标是**鼓励更短但充分的压缩**
- 在保持任务性能的前提下最小化压缩后上下文长度

两步交替进行，直到收敛。

## 自然语言梯度

ACON 用 LLM 作为优化器，通过自然语言生成"梯度"：

1. **反馈生成**：`Feedback = LLM(Feedback Instruction, H, H')` — 分析压缩时丢失了什么关键信息
2. **准则更新**：`P^{(1)} = LLM(Update Instruction, P^{(0)}, Feedback_i...)` — 聚合多条轨迹的反馈，更新压缩准则

## 特点

- **无梯度**：完全基于 LLM 的自然语言推理，不需要计算图或梯度反向传播
- **API 兼容**：可以直接用于任何 LLM API，无需 fine-tuning
- **可蒸馏**：优化后的压缩准则可以蒸馏到小模型独立部署
